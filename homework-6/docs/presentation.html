Agent 1 of 4

specification-writer → the contract

Turns broad seed requirements into a layered, traceable, execute-without-guessing spec. Its Low-Level Tasks section is literally the routing table the orchestrator reads next.

  • Reads TASKS.md + sample-transactions.json (8 rows of deliberate edge cases)
  • Emits High → Mid → Low level objectives, every low-level task traceable to a mid-level one
  • One low-level task per pipeline agent — validator, fraud, compliance, integrator, frontend
  • Writes specification only — never a line of implementation code

Agent 3 of 4

unit-test-agent → the safety net

Authors the pytest suite covering every agent plus a full-pipeline integration test through integrator.py — and must clear the mandatory ≥80% coverage gate.

  • One test module per agent + test_integrator.py end-to-end
  • Isolates from real shared/ & logs/ via PIPELINE_SHARED_ROOT + tmp_path
  • A PreToolUse hook re-runs coverage and blocks git push below 80%

The glue between agents

A file-based message protocol

Agents never call each other directly — they hand off JSON files through four folders, moved atomically with tempfile + os.replace.

input/

Envelope written per transaction_id, awaiting the validator

processing/

Transient — a message currently owned by one agent

output/

Validated / scored messages awaiting the next agent

results/

Exactly one terminal outcome + summary.json

# the standard message envelope (agents/protocol.py)
{ "message_id": "…", "timestamp": "2026-…Z", "source_agent": "validator",
  "target_agent": "fraud_detector", "message_type": "validated", "data": { … } }

Money is only ever decimal.Decimal (never float) · audit log is append-only, ISO 8601, PII-masked · every agent is idempotent.

Wrap-up

Agents all the way down

  • Spec-driven: a written contract precedes any code
  • Orchestration: one agent delegates to four specialized subagents, one at a time
  • Cooperation: runtime agents coordinate purely through files
  • Guardrails: coverage gate, PII masking, decimal money, idempotency, audit trail
